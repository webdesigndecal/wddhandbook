Introduction

Understanding how to lay things out with positioning is only the first step to really understanding CSS layouts. Through learning how to center elements on a webpage, we'll learn other methods of laying out elements. Then, we'll delve even deeper, into the modern CSS3 features, specifically flexbox. This amazing feature gives us all we need to position elements exactly the way we want to, efficiently, without the need for Bootstrap (especially if you learn CSS grids in addition to flexbox, but we will not delve into it in this semester's handbook).



Centering

There are many ways to center things, but here, we will go through three that do not use flexbox or CSS grids. Rather, they will make use of text alignment, margins & padding, positioning, and translation. These are not encouraged to be used in centering in too many cases now that flexbox and grids exist, but they're good to know and help with learning important CSS properties.


Text Alignment

Counterintuitively, text-align: center is not used only for text. It can be used to horizontally center any element with a display of inline-block or inline. There is also text-align: left and text-align: right, which align inline items to the left or right, as you would expect.


Margins & Padding

Though this section is about both margins and padding, we will only focus on margins as padding is used in nearly the same way. Margins add a specific about of space between elements. If the same amount of spacing is specified on the left and right, the element will be horizontally centered. If the same amount of spacing is specified on the top and bottom, the element will be vertically centered. For example, if the container was 200 by 200 pixels and a picture (or other content) inside of it was 50 by 50 pixels, adding the style margin: 75px, which adds margins to all sides of the picture, will center the picture horizontally and vertically. Alternatively, margin: 0 auto can be used for horizontally centering an element with a width, height, and display specified inside a parent element with a width and height.

Positioning

As we saw in the last week, using absolute positioning, we can forcibly position elements exactly where we want them to be. This method of centering would also require pixel or percentage calculations, similar to margins. If we merely position an element absolutely and then add left: 50% and top: 50%, that would make the element off-center as it pushes the edges of the element down and does not use the middle of the element as the point of reference (in other words, the top of the element will have been pushed down 50% with the element itself hanging below the 50% mark).


Therefore, if the height and the width of the element is 20% of those of its parent element, we'd have to use top: 40% and left: 40% because we take up 40% of the parent's space on both sides of the elemeent and 20% for the element itself: 40% + 20% + 40% = 100%. Note that these calculations would change yet again and become more complicated if either the child or parent elements had padding or margins, but we won't get into that. With pixels instead of percentages, the concept is the same in that spacing will have to be calculated as parts of the total.



Image source: WDD Lecture 5, Spring 2018



Translation

To use translation, we'll have to use the transform property of CSS with a value of translateX(amt) translateY(amt), where amt is replaced with a pixel or percent value. In this case, the midpoint of the element being translated is used instead of the edges like with positioning. So, adding transform: translateX(50%) translateY(50%) does center the element in this case.



Flexbox

That was a mouthful. All of the above information is a lot to take in, but luckily, if you use flexbox, you won't have to remember most (or any) of it! The name flexbox is derived from \"flexible box\", which is exactly what flexbox provides you with. These flexboxes make it super easy to handle your layouts, without the need to remember a ton of non-English words and rules within rules.


Set Display

To use a flexbox, the first thing we have to do is set the display for the element to display: flex. This tells the browser that the particular element is now a flexbox and can have its content positioned using flexbox properties and values. The first three fundamental, basic properties are flex-direction, justify-content, and align-items.


Flex Direction

We'll start with flex-direction, which can be set to column or row. Setting flex-direction: column will align elements within your flexbox element stacked one on top of another. Using row will align the contents of your container next to each other instead. Beyond simple layout differences, using column versus row values will also change the functionality of justify-content and align-items, but we'll talk about that more in a second.



Image source: https://medium.freecodecamp.org/an-animated-guide-to-flexbox



Content Alignment

To begin with, let's talk about justify-content and align-items in the context of flex-direction: row (which by the way is the default direction of flexboxes in most browsers). In this context, justify-content determines the horizontal layout of elements and align-items determines the vertical layout.


The three basic values this property can be set to are flex-start, flex-end, and center though there are more possibilities that will be explored later (catch a sneak peak in the below graphic). Intuitively, flex-start positions items at the beginning of the flexbox, which would be at the left, horizontally (justify-content: flex-start) and at the top, vertically (align-items: flex-start). The value flex-end does the opposite, pushing items to either the very right or bottom of the flexbox. To center elements horizontally in the context of flex-direction: row, use justify-content: center. To center elements vertically, use align-items: center.



Image source: https://medium.freecodecamp.org/an-animated-guide-to-flexbox



Now, I made it clear that these properties and values functioned in this way only in the context of flex-direction: row because they are reversed when flex-direction is set to column. In the context of a column flexbox, justify-content determines the vertical layout of elements and align-items determines the horizontal layout, the direct opposite of the last sentence from the first paragraph of this section. This means all the properties, values and their functions are flipped in this context. For example, justify-content: flex-start positions items at the very top of the flexbox container instead of to the very left.



Image source: WDD Lecture 4, Fall 2018



The above picture can be rotated to show how the two properties change based on the direction. The property justify-content goes across horizontally, signifying that it deals with horizontal layout when flex-direction: row is read normally and align-items goes across vertically, signifying that it deals with vertical layout. If you rotate the image so that flex-direction: column is on top and read normally, the opposite will be true, and the functionalities of the properties are reversed.


Miscellaneous Layout Values

In the earlier graphic, two commonly used values for justify-content and align-items were shown: space-between and space-around. You may have already been able to tell what they do. They are similar, but different in that space-between will put as much space between the elements of the flexbox as possible until the outer two elements touch the walls (start and end) of their flexbox container. The value space-around will put an equal amount of space around all of the elements and the two outer elements should not touch the walls of their container because there is space on either side of the element. In both settings, the amount of space between elements should be equal. As an example, in the context of flex-direction: row, if one of the values is set for justify-content, the elements will be spread out horizontally whereas they would be spread out vertically if one of the values is set for align-items.
